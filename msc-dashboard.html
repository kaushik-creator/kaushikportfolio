MSC Operational Dashboard

Designing a new home screen for Druva's MSP partners — one page that shows backup problems, storage usage, expiring licenses, customer status, and security risks across all their customers at once, instead of making them check each customer one by one.

Role
UX/UI Designer
Domain
Data Backup & Cybersecurity
Focus
Dashboards, Multi-customer Visibility, Monitoring
Type
Enterprise · B2B
Project Context

What is Druva?

Druva is a cloud backup and data security company. It helps businesses back up and protect their data across laptops, apps, and cloud systems.

What is MSC?

MSC (Managed Services Center) is Druva's portal for partner companies (MSPs) that manage backups for many customers at once, all from a single login.

What is this project about?

Before this, MSP partners logged in and saw a plain table with no useful information. This project replaced that with a real dashboard — one screen that shows what needs attention right now, across every customer they manage.

Persona: MSP Admin

Who they are

The person who manages backups, customer accounts, and security for the MSP, across every customer they support — all from one login.

What they do

  • Checks which customers had backup failures and fixes them
  • Watches for missed SLAs and storage limits across all clients
  • Tracks storage and usage trends to catch problems early, like a customer suddenly using way more storage than usual
  • Keeps an eye on which contracts or trials are about to expire, and which customers are still "active" but shouldn't be
  • Looks for customers who don't have security add-ons, as a chance to sell more
  • Checks their own admin accounts and API keys for anything outdated or risky

Goals

  • Log in once and see everything that matters, instead of digging through each customer separately
  • Know instantly which customer needs help first
  • Click on any problem and go straight to it — no extra searching

Pain Points

  • Had to open every customer one by one just to check if something was wrong
  • No easy way to see if support was already handling an issue
  • Storage and usage problems were only noticed after they became a real issue
  • Expiring contracts and unused ("zombie") customer accounts were easy to miss
  • No simple way to see who was missing security features
  • Constantly switching between screens just to get a full picture

Needs

"I want one page that shows me what's broken, what's expiring, what's growing too fast, and who I could sell more to — without clicking into every customer one at a time."

Problem Breakdown

Problem Statement

  • The old landing page was just a plain table — it didn't show any real problems or risks.
  • MSPs had to manually check each customer to find backup failures, which took time and slowed everyone down.
  • Competitors (Veeam, Acronis, Datto, Commvault) already had dashboards like this, so Druva was behind.

Design Difficulties

  • The dashboard needed to show five different kinds of information — failures, SLA/storage alerts, usage trends, account status, and security risks — without the page feeling cluttered.
  • Different types of users (admin vs. limited access) needed to see different amounts of data on the same screen.
  • Filters needed to clearly show when they were "on" so people didn't get confused about what they were looking at.
  • Every number on the dashboard needed to be clickable and lead somewhere useful, not just sit there as a stat.
  • When everything was fine, the page needed to say so clearly, instead of just looking empty or broken.
